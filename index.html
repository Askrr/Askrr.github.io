Artifact Categories:

Category 1: Software design and engineering
Artifact Overveiw:

Before enhancements, the classes Dog, Monkey, and RescueAnimal provided basic functionality for representing information about rescue animals. They included instance variables for attributes such as name, breed, age, gender, weight, acquisition details, training status, reservation status, service location, vaccination status, temperament, and special needs. The constructors and getter/setter methods facilitated the creation and manipulation of objects representing individual animals. However, the code lacked error handling, logging capabilities, and was not optimized for readability or scalability. It had also used in-program memory to add animals to lists and arrays instead establishing a connection to an external database.

After enhancements, these classes underwent significant improvements in functionality, reliability, and maintainability. The additions of error handling mechanisms, logging functionality, enhanced validation logic, and the connection to an outside database highly imploved the functionality and versatility of the program. The separation of concerns and adherence to object-oriented principles made the code more modular and easier to understand. The inclusion of the sorting functionality in the Driver class extended the capabilities of the software, providing users with additional utility for managing and analyzing animal data. Overall, the enhanced classes better aligned with industry best practices, providing a solid foundation for building and extending the rescue animal management system.

The sortAnimals method in the Driver class further showcases algorithmic principles by implementing sorting functionality for animal data based on different criteria.

Category 2: Algorithms and data structure
Artifact Overveiw:

The Driver.java class, originating from the IT145 Application Development coursework, serves as the entry point for executing algorithms and data structures related to animal rescue management. Originally, the Driver class provided basic functionality for sorting animal data using the quicksort algorithm and performing binary search operations to locate specific animal records. However, the original implementation lacked comprehensive error handling mechanisms and optimization techniques. Through enhancements made in Milestone Two, the Driver class underwent significant improvements, including the optimization of sorting and searching algorithms, integration of error handling mechanisms, and enhancement of code readability. These enhancements transformed the Driver class into a well-rounded and efficient component of the animal rescue management system.

Category 3: Databases
Artifact Overveiw:

The AnimalRepository.java class was developed as a crucial component of the database management system for the animal rescue management application. This class was created from scratch to facilitate seamless interaction with the MySQL database, allowing for efficient handling of animal data transactions. Initially conceived during the development phase of the project, the AnimalRepository class underwent significant enhancements to optimize database operations, integrate error handling mechanisms, and improve overall code quality. These enhancements transformed the AnimalRepository class into a reliable module essential for ensuring the integrity and efficiency of database interactions within the animal rescue management system.
